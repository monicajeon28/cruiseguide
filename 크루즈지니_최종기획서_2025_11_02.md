# 크루즈 가이드 '지니(Genie)' - 최종 기획서

> **작성일**: 2025년 11월 2일  
> **버전**: 2.0 (현실적 범위 조정)  
> **목표**: 50대 이상 고객이 크루즈 가이드 없이 안전하고 편안한 여행

---

## 🎯 프로젝트 비전

### 핵심 가치
**"여행 예약부터 귀가까지, 걱정 없는 크루즈 여행의 동반자"**

50대 이상 고령 고객도 쉽게 사용할 수 있는 AI 크루즈 가이드로,
복잡한 크루즈 여행을 30초 만에 시작하고, 능동적으로 보호받으며,
음성으로 편하게 소통하는 혁신적인 서비스

---

## 🎯 핵심 목표 3가지

### 1. 초간단 시작 (30초 온보딩)
- **현재**: 여행사에서 10분 이상 걸리는 복잡한 정보 입력
- **지니**: 예약번호 + 출발날짜 = **30초 완료**
- **효과**: 온보딩 장벽 90% 감소

### 2. 능동적 보호 (출항 놓침 0건)
- **현재**: 고객이 시간 체크, 자주 놓침
- **지니**: AI가 먼저 알려줌 (출항 1시간 전 푸시 알림)
- **효과**: 출항 놓침 사고 **0건** 보장

### 3. 시니어 친화 (음성 중심)
- **현재**: 작은 글씨, 복잡한 메뉴, 타이핑 필요
- **지니**: 음성 출력/입력, 큰 글씨, 단순한 UI
- **효과**: 50대+ 만족도 **90%+**

## 🏗️ 시스템 아키텍처

### 기술 스택
- **Frontend**: Next.js 14 (App Router), React 18, TypeScript
- **Backend**: Next.js API Routes, Prisma ORM
- **Database**: SQLite (개발) → PostgreSQL (운영)
- **AI**: Google Gemini 2.5 Flash
- **인증**: Iron Session
- **푸시 알림**: Web Push API (VAPID)
- **음성**: Web Speech API (TTS/STT)
- **스타일**: Tailwind CSS

### 핵심 시스템 4개

#### 1. 자동화 시스템 (스케줄러)
```
매 10분마다: Proactive Engine (5가지 알림)
매일 자정: Trip Status Updater (여행 상태 전환)
매일 새벽 2시: Hibernation Checker (90일 동면)
매주 월요일 10시: Reactivation Sender (재활성화 알림)
```

#### 2. 데이터베이스 (17개 모델)
```
사용자: User, Session, LoginLog, PasswordEvent
여행: Trip, Itinerary, CruiseProduct, VisitedCountry
도구: Expense, ChecklistItem, TravelDiaryEntry
AI: ChatHistory, KnowledgeBase
알림: PushSubscription, NotificationLog
CMS: CmsNotificationTemplate, TripFeedback
```

#### 3. API (60+개 엔드포인트)
```
인증: 로그인, 프로필, 세션
여행: 자동 생성, 일정, 상태 관리
채팅: AI 대화, 히스토리
알림: 푸시 구독, 발송
데이터: 가계부, 체크리스트
CMS: 템플릿, 상품 관리
관리자: 대시보드, 통계, 긴급 공지
```

#### 4. 보안 시스템 (5중 방어)
```
1. CSRF 보호 (모든 POST/PUT/DELETE)
2. Rate Limiting (IP 기반)
3. 세션 관리 (30일 자동 만료)
4. 보안 헤더 (7개)
5. 통합 로깅 (모든 접근 기록)
```

### 2. 능동적 보호자 (Proactive Engine)

#### 5가지 자동 트리거

##### 트리거 1: D-Day 알림
```
시점: D-7, D-1 (오전 9시)
중요도: MEDIUM
메시지: 
  D-7: "출발 7일 전! 전자기기 충전 확인하세요"
  D-1: "내일 출발! 여권과 집합시간 최종 확인하세요"
```

##### 트리거 2: 승선 안내
```
시점: 승선 3시간 전
중요도: HIGH
메시지: "터미널로 이동할 시간입니다! 🚢"
액션: Google Maps 링크 제공
```

##### 트리거 3: 기항지 도착
```
시점: 도착 1시간 전
중요도: MEDIUM
메시지: "[기항지명] 도착 1시간 전! 여권을 챙기세요 📍"
```

##### 트리거 4: 🚨 귀선 경고 (핵심!)
```
시점: 출항 1시간 전
중요도: ⚠️ CRITICAL
메시지: "⚠️ 긴급! 출항 1시간 전! 지금 바로 배로 돌아오세요!"
특징: 
  - requireInteraction: true (확인할 때까지 유지)
  - 진동: [200, 100, 200]
  - 액션 버튼: [확인하기, 닫기]
효과: 출항 놓침 0건 보장
```

##### 트리거 5: 하선 준비
```
시점: 하선 1시간 전
중요도: HIGH
메시지: "하선 준비! 여권 회수 및 짐을 챙기세요 🏠"
```

#### 중복 방지 시스템
```typescript
eventKey = `${userId}-${tripId}-${type}-${day}-${itineraryId}`
// 예: "123-456-BOARDING_WARNING-2-789"

- NotificationLog 테이블에서 eventKey 존재 확인
- 이미 발송된 알림은 재발송 안 함
- DB 유니크 제약으로 2중 보호
```
