# 크루즈 웹사이트 API 분석 가이드

## 1단계: Chrome 개발자 도구로 API 찾기

### cruisedot.co.kr 분석

1. **Chrome에서 사이트 열기**
   - https://www.cruisedot.co.kr/ 접속

2. **개발자 도구 열기**
   - `F12` 키 누르기 또는
   - 우클릭 → "검사" 클릭

3. **Network 탭으로 이동**
   - 개발자 도구 상단의 "Network" 탭 클릭
   - 왼쪽 상단의 빨간 녹화 버튼이 켜져 있는지 확인

4. **필터 설정**
   - "Fetch/XHR" 버튼 클릭 (API 호출만 보기)

5. **페이지 새로고침**
   - `Ctrl + R` 또는 `F5` 키로 페이지 새로고침
   - 네트워크 요청들이 목록에 나타남

6. **API 찾기**
   다음 중 하나를 찾아보세요:
   - `/api/products` 또는 `/api/cruise`
   - `/api/list` 또는 `/api/search`
   - 상품 목록 페이지로 이동하면 자동으로 API 호출됨

7. **API 응답 확인**
   - 찾은 API를 클릭
   - "Response" 탭에서 JSON 데이터 확인
   - "Headers" 탭에서 Request URL 복사

### wcruisenco.kr 분석

위의 1-7단계를 https://www.wcruisenco.kr/ 에서도 반복합니다.

## 💡 팁

### API를 못 찾겠다면?

1. **상품 목록 페이지로 이동**
   - 메뉴에서 "크루즈 상품", "여행 상품" 등 클릭
   - 상품들이 로드될 때 API가 호출됨

2. **검색 기능 사용**
   - 사이트 내 검색 기능 사용
   - 검색 결과 로드 시 API 호출됨

3. **필터/정렬 사용**
   - 가격순, 날짜순 정렬 버튼 클릭
   - 필터 변경 시 API가 다시 호출됨

### 서버 사이드 렌더링(SSR)인 경우

만약 API 호출이 전혀 보이지 않는다면, 이 웹사이트는 서버에서 HTML을 생성해서 보내는 방식일 수 있습니다. 이 경우:

1. **Document 탭 확인**
   - HTML 파일을 클릭
   - "Preview" 탭에서 페이지 구조 확인
   - 상품 데이터가 HTML에 직접 포함되어 있는지 확인

2. **대안**
   - HTML 파싱 방식으로 스크래핑 (Cheerio 사용)
   - 또는 여행사에 API 제공 요청
