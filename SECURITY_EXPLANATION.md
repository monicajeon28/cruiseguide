# 보안 기능 상세 설명 문서

## ✅ 구현 완료된 기능들 (현재 작동 중)

### 1. **웹 브라우저에서 우클릭으로 이미지 저장 방지** ✅
- **구현 방법**: `contextmenu` 이벤트 차단
- **코드 위치**: `SecurityProtection.tsx` 14-17줄
- **작동 여부**: ✅ 완전히 차단됨
- **설명**: 마우스 우클릭 시 나타나는 컨텍스트 메뉴를 완전히 차단합니다.

### 2. **이미지 드래그 앤 드롭 방지** ✅
- **구현 방법**: 
  - `dragstart` 이벤트 차단 (JavaScript)
  - `draggable="false"` 속성 추가
  - CSS `user-drag: none` 적용
- **코드 위치**: 
  - `SecurityProtection.tsx` 26-29줄 (이벤트)
  - `SecurityProtection.tsx` 143줄 (속성)
  - `globals.css` 61-65줄 (CSS)
- **작동 여부**: ✅ 완전히 차단됨
- **설명**: 이미지를 드래그하여 다른 곳으로 옮기는 것을 방지합니다.

### 3. **텍스트 선택 및 복사 방지** ✅
- **구현 방법**:
  - `selectstart` 이벤트 차단
  - CSS `user-select: none` 적용
  - `copy`, `cut` 이벤트 차단
- **코드 위치**:
  - `SecurityProtection.tsx` 20-23줄 (선택 방지)
  - `SecurityProtection.tsx` 112-115줄 (복사 방지)
  - `globals.css` 32-35줄 (CSS)
- **작동 여부**: ✅ 완전히 차단됨
- **설명**: 텍스트를 선택하거나 복사하는 것을 방지합니다. (입력 필드는 예외)

### 4. **일반적인 키보드 단축키 방지** ✅
- **구현 방법**: `keydown` 이벤트에서 특정 키 조합 차단
- **차단되는 단축키**:
  - F12 (개발자 도구)
  - Ctrl+Shift+I (개발자 도구)
  - Ctrl+Shift+J (콘솔)
  - Ctrl+Shift+C (요소 검사)
  - Ctrl+U (소스 보기)
  - Ctrl+S (저장)
  - Ctrl+P (인쇄)
  - Ctrl+Shift+P (명령 팔레트)
- **코드 위치**: `SecurityProtection.tsx` 32-86줄
- **작동 여부**: ✅ 대부분 차단됨 (일부 브라우저 제한 있음)

### 5. **모바일에서 이미지 롱프레스 방지** ✅
- **구현 방법**:
  - `touchstart` 이벤트 감지
  - 500ms 이상 누르면 차단
  - CSS `-webkit-touch-callout: none` 적용
- **코드 위치**:
  - `SecurityProtection.tsx` 95-109줄 (이벤트)
  - `globals.css` 38줄, 60줄 (CSS)
- **작동 여부**: ✅ 완전히 차단됨
- **설명**: 모바일에서 이미지를 길게 눌렀을 때 나타나는 저장 메뉴를 방지합니다.

### 6. **동적으로 추가되는 이미지에도 자동 적용** ✅
- **구현 방법**: `MutationObserver` 사용
- **코드 위치**: `SecurityProtection.tsx` 150-183줄
- **작동 여부**: ✅ 완전히 작동함
- **설명**: 페이지 로드 후 JavaScript로 추가되는 이미지에도 자동으로 보안 설정이 적용됩니다.

## ❌ 기술적 제약으로 차단 불가능한 항목들

### 1. **운영체제 레벨 스크린샷** ❌
**왜 차단 불가능한가?**
- **기술적 이유**: 웹 브라우저는 운영체제의 하드웨어 레벨 기능에 접근할 수 없습니다.
- **구체적 예시**:
  - **iOS**: 볼륨 업 + 전원 버튼 동시 누름
  - **Android**: 전원 버튼 + 볼륨 다운 버튼 동시 누름
  - **Windows**: Print Screen 키, Windows + Shift + S
  - **Mac**: Command + Shift + 3/4
- **왜 막을 수 없나?**
  - 이러한 기능은 **운영체제 커널 레벨**에서 작동합니다.
  - 웹 브라우저는 **애플리케이션 레벨**에서만 작동하므로, OS 기능을 제어할 수 없습니다.
  - 브라우저가 OS의 스크린샷 기능을 차단하려면, 브라우저 자체가 OS 권한을 가져야 하는데 이는 보안상 위험하고 불가능합니다.

### 2. **하드웨어 스크린샷 도구** ❌
**왜 차단 불가능한가?**
- **기술적 이유**: 외부 하드웨어나 소프트웨어는 브라우저의 제어 범위 밖입니다.
- **구체적 예시**:
  - 외장 캡처 카드
  - 스크린 레코더 소프트웨어 (OBS, Bandicam 등)
  - 가상 머신의 스크린샷 기능
- **왜 막을 수 없나?**
  - 이러한 도구들은 **브라우저와 독립적으로** 작동합니다.
  - 화면의 **픽셀 데이터**를 직접 읽어오므로, 웹 페이지의 JavaScript로는 감지하거나 차단할 수 없습니다.

### 3. **브라우저 확장 프로그램** ❌
**왜 차단 불가능한가?**
- **기술적 이유**: 브라우저 확장 프로그램은 웹 페이지보다 더 높은 권한을 가집니다.
- **구체적 예시**:
  - 스크린샷 확장 프로그램
  - 이미지 다운로더 확장 프로그램
  - 개발자 도구 확장 프로그램
- **왜 막을 수 없나?**
  - 확장 프로그램은 **브라우저 API**에 직접 접근할 수 있습니다.
  - 웹 페이지는 확장 프로그램의 존재를 감지할 수 있지만, **차단할 권한이 없습니다**.
  - 확장 프로그램은 웹 페이지의 JavaScript 실행 전에 작동할 수 있습니다.

### 4. **개발자 도구를 통한 접근** ⚠️ (일부만 방지)
**왜 완전히 차단 불가능한가?**
- **기술적 이유**: 개발자 도구는 브라우저의 핵심 기능이며, 완전히 차단하면 웹 개발이 불가능합니다.
- **현재 구현된 차단**:
  - ✅ F12 키 차단
  - ✅ Ctrl+Shift+I 차단
  - ✅ Ctrl+Shift+J 차단
  - ✅ Ctrl+U (소스 보기) 차단
- **여전히 가능한 방법들**:
  - ❌ 브라우저 메뉴에서 직접 개발자 도구 열기
  - ❌ 확장 프로그램을 통한 개발자 도구 접근
  - ❌ 브라우저 설정에서 개발자 도구 활성화
  - ❌ 다른 키보드 단축키 (브라우저마다 다름)
- **왜 완전히 막을 수 없나?**
  - 개발자 도구는 **웹 표준의 일부**입니다.
  - 브라우저 제조사들은 개발자 도구를 완전히 차단하는 것을 허용하지 않습니다 (웹 접근성 및 개발 편의성 때문).
  - 개발자 도구를 통해 DOM을 조작하면 모든 보안 설정을 우회할 수 있습니다.

## 🔍 추가 개선 가능한 항목들

현재 구현에서 더 강화할 수 있는 부분:

1. **개발자 도구 감지**: 개발자 도구가 열렸는지 감지하여 경고 표시 (완전 차단은 불가능하지만 경고 가능)
2. **스크린샷 시도 감지**: 일부 브라우저에서 스크린샷 시도를 감지할 수 있음 (제한적)
3. **워터마크 추가**: 스크린샷이 찍혀도 추적 가능하도록 투명 워터마크 추가
