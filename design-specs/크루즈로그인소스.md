이제 로그인 시스템을 최종적으로 완성하자. 사용자의 이름을 기억하고 개인화된 서비스를 제공해야 하므로, 이름, 핸드폰 번호, 비밀번호를 모두 사용해야 해.

1.  **로그인 페이지 UI 수정 (`src/app/login/page.tsx`)**:
    -   `src/app/login/page.tsx` 파일을 열어줘.
    -   화면 중앙에 있는 카드에 **'이름', '핸드폰 번호', '비밀번호'** 세 개의 입력창이 모두 보이도록 구성해줘.
    -   '비밀번호' 입력창은 `type="password"` 속성을 가져서, 입력 내용이 점(•)으로 보이도록 마스킹 처리를 해야 해.

2.  **백엔드 인증 로직 업그레이드 (`/api/auth/login`)**:
    -   `/api/auth/login` 백엔드 로직을 수정하여, `name`, `phone`, `password` 세 개의 값을 모두 받도록 변경해줘.
    -   로직은 아래 순서대로 작동해야 해:
        1.  먼저, 입력된 `password`가 **우리가 정한 비밀번호('3800')와 일치하는지 확인**해줘. 일치하지 않으면 "비밀번호가 올바르지 않습니다." 라는 메시지와 함께 로그인을 거부해줘.
        2.  비밀번호가 일치하면, Prisma를 사용하여 DB에서 `phone` 번호가 일치하는 사용자가 있는지 찾아줘.
        3.  **만약 사용자가 이미 존재한다면 (기존 고객):** 바로 로그인시키고 JWT를 발급해줘.
        4.  **만약 사용자가 존재하지 않는다면 (신규 고객):** 입력받은 `name`과 `phone`, 그리고 비밀번호 '3800'을 사용하여 `User` 테이블에 **새로운 사용자를 생성(회원가입)**한 뒤, 로그인시키고 JWT를 발급해줘.
