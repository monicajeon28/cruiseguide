### Phase 2: 핵심 안전 기능 (1주)

**난이도**: ⭐⭐⭐☆☆ (중급)

**어려운 이유**:
1. **GPS 권한 처리**: 브라우저별 다르게 작동
2. **실시간 카운트다운**: `setInterval` 관리, 메모리 누수 위험
3. **Google Maps URL 생성**: 좌표 변환, 파라미터 이해
4. **Itinerary 데이터 구조**: 출항 시간, 항구 좌표 어디서 가져오나?

**현실**:
```typescript
// 당신이 만든 코드
const departureTime = itinerary.departure; // ??? 이게 어디 있나?

// 실제로 필요한 것
const departureTime = itinerary.times?.departure || itinerary.departureTime || trip.endTime;
// → 데이터 구조를 완전히 이해해야 함
```

**실제 소요 시간**: 계획 1주 → **실제 2주**

### Phase 3: RAG 시스템 구축 (2-3주) ⚠️ **가장 어려움**

**난이도**: ⭐⭐⭐⭐⭐ (고급-전문가급)

#### 왜 이렇게 어려운가?

**1. PostgreSQL + pgvector 설정**
```
당신의 현재 환경: SQLite (파일 기반 DB)
필요한 환경: PostgreSQL + pgvector 확장

이것만으로도:
- PostgreSQL 설치 (로컬 or 클라우드)
- Supabase or Neon 같은 서비스 선택
- Prisma 연결 설정 변경
- pgvector 확장 활성화
- 환경 변수 설정
→ 초보자에게는 한 달짜리 작업
```

**2. 임베딩 생성 스크립트**
```typescript
// 당신의 계획서에 있는 코드
const embedding = await googleAI.embeddings.create({
  model: 'embedding-001',
  input: chunkText,
});
// → 이 API 호출이 실제로 작동하려면
// → 1. API 키 설정
// → 2. Rate Limiting 처리
// → 3. 에러 핸들링
// → 4. 비용 관리 (임베딩은 유료)
```

**3. 벡터 검색 쿼리**
```sql
-- 당신은 이런 SQL을 직접 작성해야 함
SELECT content, title, 
       1 - (embedding <=> $1::vector) AS similarity
FROM "KnowledgeBase"
ORDER BY similarity DESC
LIMIT 5;
-- → pgvector 문법 이해 필요
-- → Prisma Raw Query로 감싸야 함
```

**4. 콘텐츠 작성 (당신만 할 수 있는 것)**
```
✓ 전체 크루즈 여행 가이드 작성
✓ 50개 이상 FAQ 작성
✓ 단계별 상세 설명 (친절하게)
→ 이것은 당신만 할 수 있는 일
→ 시간: 최소 1-2개월
```

**현실적 평가**:
- ✅ **당신만 할 수 있는 것**: 콘텐츠 작성
- ❌ **거의 불가능**: 기술적 구현 (RAG 시스템)

**해결 방안**: 
- Option 1: 전문가 협업 (개발자 고용)
- Option 2: 간단한 버전으로 시작 (벡터 DB 없이 키워드 매칭)
