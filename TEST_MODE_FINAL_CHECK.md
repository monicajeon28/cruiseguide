## 🎯 테스트 시나리오

### 시나리오 1: 신규 사용자 로그인
1. `/login-test` 접속
2. 이름, 연락처, 비밀번호(1101) 입력
3. 로그인 성공 → `/chat-test`로 이동
4. 자동 온보딩 확인 (SAMPLE-MED-001, 출발일 3일 전)
5. 오늘의 브리핑에 온보딩 정보 표시 확인
6. D-3 메시지 팝업 표시 확인

### 시나리오 2: 네비게이션 테스트
1. `/chat-test`에서 하단 네비게이션 바의 "도구함" 클릭
2. `/tools-test`로 이동 확인
3. "체크리스트" 클릭 → `/checklist-test`로 이동 확인
4. "뒤로가기" 클릭 → `/tools-test`로 이동 확인
5. 하단 네비게이션 바의 "홈" 클릭 → `/chat-test`로 이동 확인

### 시나리오 3: 내 정보 D-day 메시지 확인
1. `/profile-test` 접속
2. "지니의 여행 준비 가이드" 섹션 확인
3. 현재 D-day에 맞는 메시지가 "지금 봐야 할 메시지"로 표시되는지 확인
4. 앞으로 봐야 할 메시지와 이미 봤던 메시지가 올바르게 구분되는지 확인

### 시나리오 4: 로그아웃 테스트
1. `/chat-test`에서 상단 카운트다운 배너의 "로그아웃" 버튼 클릭
2. `/login-test`로 이동 확인
3. 다시 로그인 시도 → 정상 작동 확인

## ✅ 최종 결론

모든 주요 기능이 정상적으로 작동하며, 테스트 모드와 일반 모드가 완전히 분리되어 있습니다. 발견된 하드코딩된 링크는 일반 모드에서만 사용되므로 테스트 모드에는 영향을 주지 않습니다.
